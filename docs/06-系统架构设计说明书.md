# 系统架构设计说明书

## 1. 逻辑架构

```
┌─────────────────────────────────────────────┐
│ 表现层：Vue3 公众门户 + Vue3 管理后台        │
├─────────────────────────────────────────────┤
│ 网关层：Nginx 反向代理 / 静态资源            │
├─────────────────────────────────────────────┤
│ 应用层：Spring Boot REST API                │
│   auth | content | portal | file | ai       │
├─────────────────────────────────────────────┤
│ 数据层：MySQL 8.0 | Redis 7 | 文件存储       │
└─────────────────────────────────────────────┘
```

## 2. 部署架构（Docker Compose）

- nginx:80 → 前端静态 + /api 代理  
- backend:8080 → Spring Boot  
- mysql:3306、redis:6379  

## 3. 技术选型

| 技术 | 选型理由 |
|------|----------|
| Spring Boot 3.2 | 企业主流、生态完善 |
| MyBatis-Plus | 高效 CRUD、分页 |
| Vue 3 + Element Plus | 课程要求、组件丰富 |
| Redis | 统计缓存、性能加分 |
| SpringAI | 课程要求、健康问答创新点 |
| JWT | 无状态认证，前后端分离 |

## 4. 安全设计

- 密码 BCrypt 存储  
- JWT 鉴权，后台接口 RBAC（ADMIN/EDITOR）  
- 公众接口只读，管理接口需 Token  
- 文件上传类型与大小限制（50MB）  
- MyBatis-Plus 参数化防 SQL 注入  

## 5. 与「大数据」定位说明

本期实现**门户与内容治理层**，作为健康大数据平台的统一入口；后续可对接数据中台 API 扩展「应用中心」数据服务。
