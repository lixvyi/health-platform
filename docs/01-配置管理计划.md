# 配置管理计划

**项目名称**：健康大数据应用创新研发中心门户系统  
**项目组**：3人（成员A-后端/项目经理，成员B-前端，成员C-测试/数据库）  
**版本**：V1.0

## 1. 配置库

- 工具：Git
- 远程仓库：`origin`（由项目组在 GitHub/Gitee 创建）
- 本地根目录：`platform/`

## 2. 目录结构

```
platform/
├── docs/
├── health-portal-backend/
├── health-portal-frontend/
├── docker/
├── sql/
└── tests/jmeter/
```

## 3. 分支策略（GitFlow 简化）

| 分支 | 用途 |
|------|------|
| main | 发布稳定版，仅合并 develop |
| develop | 日常集成分支 |
| feature/* | 功能开发，如 feature/news-crud |
| fix/* | 缺陷修复 |

## 4. 命名规范

- 分支：`feature/模块-功能`，如 `feature/portal-home`
- 提交：Conventional Commits，如 `feat(portal): 首页轮播接口`
- 文档：`docs/序号-文档名.md`

## 5. 提交与评审

- 小步提交，每次提交对应一个完整小功能
- feature 合并 develop 前需另一名成员 Code Review
- 禁止直接向 main 推送

## 6. 配置项

| 类型 | 说明 |
|------|------|
| 源代码 | backend、frontend |
| 数据库脚本 | sql/init.sql |
| 部署配置 | docker/ |
| 项目文档 | docs/ |
| 测试资产 | tests/jmeter/ |

## 7. 敏感配置

- `AI_API_KEY`、数据库密码通过环境变量注入
- 禁止将密钥提交至 Git（.gitignore 已配置 .env）
