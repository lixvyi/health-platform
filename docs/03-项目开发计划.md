# 项目开发计划

**项目名称**：健康大数据应用创新研发中心门户系统  
**实训周期**：12 工作日  
**团队规模**：3 人

## 1. WBS 工作分解

| WBS | 任务 | 负责人 | 产出 |
|-----|------|--------|------|
| 1.0 | 项目启动与先启 | A | 配置库、SRS、计划 |
| 1.1 | 需求基线评审 | 全员 | 用例模型 |
| 2.0 | 精化阶段 | | |
| 2.1 | 架构与数据库设计 | A+C | 架构文档、PDM |
| 2.2 | 原型设计 | B | Axure 原型 |
| 2.3 | 测试计划 | C | 测试计划/用例 |
| 3.0 | 构建阶段 | | |
| 3.1 | 后端框架与认证 | A | Spring Boot 脚手架 |
| 3.2 | 内容管理 API | A | CRUD 接口 |
| 3.3 | 公众门户前端 | B | Vue 门户 |
| 3.4 | 管理后台前端 | B | Vue Admin |
| 3.5 | Redis/SpringAI | A | 缓存与 AI |
| 3.6 | 单元测试与集成 | C | JUnit、联调 |
| 4.0 | 产品化 | | |
| 4.1 | 部署与压测 | C | Docker、JMeter |
| 4.2 | 文档与答辩 | 全员 | 用户手册、PPT |

## 2. 三人分工

| 成员 | 角色 | 主要职责 |
|------|------|----------|
| **A** | 项目经理/后端负责人 | Git、计划、后端全部模块、SpringAI、架构/概要设计 |
| **B** | 前端负责人 | 公众门户、管理后台、Axure 原型、界面设计 |
| **C** | 测试/DBA | PowerDesigner PDM、建库、测试计划/用例、JUnit、JMeter |

## 3. 里程碑

| 里程碑 | 阶段 | 交付物 |
|--------|------|--------|
| M1 | 先启结束 | SRS、配置计划、Git 仓库 |
| M2 | 精化结束 | 架构设计、PDM、原型、测试计划 |
| M3 | 构建结束 | 可运行系统、单元测试 |
| M4 | 产品化 | 部署包、答辩 PPT、实训报告 |

## 4. 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
