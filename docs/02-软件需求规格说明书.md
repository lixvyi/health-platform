# 软件需求规格说明书（SRS）

**项目编号**：11号  
**项目名称**：健康大数据应用创新研发中心门户系统  
**版本**：V1.0

## 1. 引言

### 1.1 编写目的

明确门户系统功能需求、非功能需求及用例模型，作为设计、开发与测试的依据。

### 1.2 项目背景

中心依托卫健委、大数据局等数据资源，需建设对外门户，发布新闻、政策、健康知识及应用入口，并提供后台内容管理能力。

### 1.3 术语

| 术语 | 说明 |
|------|------|
| CMS | 内容管理系统 |
| JWT | JSON Web Token 无状态认证 |
| 门户 | 面向公众的 Web 展示站点 |

## 2. 总体描述

### 2.1 用户角色

| 角色 | 描述 |
|------|------|
| 访客 | 浏览门户各栏目，使用 AI 问答 |
| 内容编辑 | 维护新闻、公告、政策、知识库 |
| 系统管理员 | 用户权限、轮播、应用、站点配置 |

### 2.2 功能模块（文档要求全覆盖）

1. 门户首页  
2. 新闻中心  
3. 通知公告  
4. 应用中心  
5. 卫生政策  
6. 健康知识库  
7. 关于我们  
8. 门户系统内容管理

## 4. 用例说明（节选）

### UC-01 管理员登录

- **参与者**：管理员/编辑  
- **前置条件**：账号已分配  
- **主成功场景**：输入用户名密码 → 校验通过 → 返回 JWT → 进入后台  
- **扩展**：密码错误提示；账号禁用拒绝登录  

### UC-02 浏览门户首页

- **参与者**：访客  
- **主成功场景**：打开首页 → 展示轮播、简介、新闻摘要、公告摘要、应用入口  

### UC-03 浏览新闻列表与详情

- **参与者**：访客  
- **主成功场景**：进入新闻中心 → 分页列表 → 点击标题 → 查看详情，阅读量+1  

### UC-04 管理新闻内容

- **参与者**：编辑  
- **主成功场景**：登录后台 → 新增/编辑/删除新闻 → 设置发布状态 → 前台可见  

### UC-05 管理通知公告（UC-06 卫生政策、UC-07 健康知识库结构同 UC-04）

### UC-08 管理应用中心入口

- **主成功场景**：后台维护应用名称、图标、链接、排序 → 前台应用中心展示  

### UC-09 管理关于我们

- **主成功场景**：后台编辑标题与富文本 → 前台关于我们页面展示  

### UC-10 AI 健康知识问答

- **参与者**：访客  
- **主成功场景**：输入问题 → SpringAI 返回答案 → 记录会话历史  
- **约束**：不提供医疗诊断，仅科普参考  

### UC-11 数据统计看板

- **参与者**：管理员  
- **主成功场景**：后台查看看板 → ECharts 展示各栏目内容数量  

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 首页接口 50 并发响应 &lt; 2s（JMeter 验证） |
| 安全 | JWT + RBAC；密码 BCrypt；防 SQL 注入 |
| 可用性 | 7×24 Web 访问；Docker 部署 |
| 可维护 | 分层架构；接口文档 Knife4j |
